## Descripción

Este proyecto se centra en la enseñanza de la programación utilizando el lenguaje PSeInt. Proporciona ejercicios prácticos y ejemplos de código para ayudar a los estudiantes a comprender los conceptos básicos de la programación.

## Instrucciones de instalación

1. Descarga e instala PSeInt desde [sitio web oficial](https://www.pseint.net/).
2. Clona o descarga este repositorio en tu máquina local.
3. Abre los archivos `.psc` en PSeInt para acceder a los ejercicios y ejemplos.

## Día 1: Introducción a la programación y PSeint

### Paso 1: Breve introducción a la programación y su importancia

Explica qué es la programación y por qué es importante en la actualidad.
Menciona cómo la programación se utiliza en diversas áreas, como el desarrollo de software, la inteligencia artificial, la ciencia de datos y la creación de sitios web.
Destaca las habilidades que se pueden desarrollar a través de la programación, como el pensamiento lógico, la resolución de problemas y la creatividad.

### Paso 2: Conceptos básicos de algoritmos y diagramas de flujo

Define qué es un algoritmo y cómo se relaciona con la programación.
Explica que un algoritmo es una secuencia ordenada de pasos que describe cómo resolver un problema.
Introduce los conceptos básicos de los diagramas de flujo, que son representaciones visuales de los algoritmos.
Muestra ejemplos de diagramas de flujo simples y explica los símbolos utilizados, como las formas de inicio y fin, las decisiones y las acciones.

### Paso 3: Presentación de PSeint y su entorno de desarrollo

Describe qué es PSeint y cómo se utiliza como herramienta para aprender a programar.
Explica que PSeint es un entorno de programación en español que permite escribir y ejecutar algoritmos utilizando un pseudocódigo.
Menciona las ventajas de utilizar PSeint para principiantes, como su sintaxis sencilla y la posibilidad de enfocarse en los conceptos de programación antes de aprender un lenguaje de programación específico.
Muestra capturas de pantalla del entorno de desarrollo de PSeint y señala las áreas clave, como el editor de código, la ventana de resultados y la barra de herramientas.

### Paso 4: Ejercicios prácticos

Proporciona ejercicios sencillos para que los participantes practiquen la escritura de algoritmos en PSeint.
Comienza con ejercicios que impliquen pasos secuenciales simples, como sumar dos números o imprimir un mensaje en pantalla.
A medida que los participantes ganen confianza, puedes incluir ejercicios que involucren decisiones condicionales y bucles básicos.
Anima a los participantes a comentar su código y explicar sus algoritmos paso a paso.
Recuerda fomentar un ambiente de participación activa, resolviendo las dudas de los participantes y motivándolos a practicar y explorar por sí mismos. Al final del día, puedes asignar tareas para la práctica adicional y repasar los conceptos clave. ¡Buena suerte con tu primer día de curso!

## Día 2: Variables y tipos de datos

### Paso 1: Declaración y uso de variables en PSeint

Explica qué es una variable y su importancia en la programación.
Muestra cómo se declara una variable en PSeint utilizando la sintaxis adecuada, por ejemplo: definir nombre como cadena.
Explica cómo asignar un valor a una variable utilizando el operador de asignación <-, por ejemplo: nombre <- "Juan".
Muestra ejemplos de uso de variables en algoritmos simples, como almacenar datos ingresados por el usuario o realizar cálculos utilizando variables.

### Paso 2: Tipos de datos básicos: enteros, decimales, caracteres y booleanos

Introduce los tipos de datos básicos en PSeint y explica su uso.
Enumera los tipos de datos más comunes, como enteros, decimales, caracteres y booleanos.
Describe las características y rangos de cada tipo de dato.
Muestra ejemplos de declaración de variables para cada tipo de dato, por ejemplo: definir edad como entero, definir salario como decimal, definir inicial como caracter, definir esValido como booleano.

### Paso 3: Conversión entre tipos de datos

Explica la necesidad y el proceso de conversión entre tipos de datos en PSeint.
Muestra cómo convertir una variable de un tipo de dato a otro utilizando las funciones de conversión disponibles en PSeint.
Proporciona ejemplos de conversión entre tipos de datos, como convertir un entero a decimal o convertir un caracter a entero.
Advierte sobre la importancia de tener en cuenta los límites y las posibles pérdidas de información al realizar conversiones entre tipos de datos.

### Paso 4: Ejercicios prácticos

Proporciona ejercicios que permitan practicar la declaración y el uso de variables con diferentes tipos de datos en PSeint.
Incluye ejercicios que involucren operaciones matemáticas simples utilizando variables de tipo entero y decimal.
Crea ejercicios que requieran la entrada de datos por parte del usuario y su almacenamiento en variables.
Incluye ejercicios que demuestren la conversión entre tipos de datos, como convertir un número entero a un caracter y viceversa.
Recuerda alentar a los participantes a comentar su código y explicar cada paso de sus algoritmos. También es importante proporcionar retroalimentación y resolver las dudas que puedan surgir durante los ejercicios prácticos. Al final del día, repasa los conceptos clave y asigna tareas de práctica adicional para reforzar el aprendizaje. ¡Buena suerte con el segundo día de curso!

## Día 3: Estructuras de control condicionales

### Paso 1: Uso de la estructura "Si-Entonces-Sino" en PSeint

Explica la estructura básica de la declaración "Si-Entonces-Sino" en PSeint.
Muestra la sintaxis adecuada, por ejemplo:
```
Si <condición> Entonces
  <bloque de código si la condición es verdadera>
Sino
  <bloque de código si la condición es falsa>
FinSi
```
Ejemplifica el uso de la estructura "Si-Entonces-Sino" en situaciones cotidianas, como tomar decisiones basadas en condiciones.

### Paso 2: Condiciones simples y compuestas

Explica las condiciones simples utilizando operadores relacionales, como igualdad (=), desigualdad (<>), mayor que (>), menor que (<), mayor o igual que (>=), y menor o igual que (<=).
Muestra ejemplos de condiciones simples, como verificar si un número es par o impar, o si un valor es mayor o menor que otro.
Introduce las condiciones compuestas utilizando operadores lógicos, como el operador Y (Y), el operador O (O), y el operador NO (NO).
Ejemplifica el uso de condiciones compuestas para evaluar múltiples condiciones simultáneamente.

### Paso 3: Ejercicios prácticos

Proporciona ejercicios prácticos para que los participantes practiquen el uso de estructuras de control condicionales en PSeint.
Incluye ejercicios que involucren la toma de decisiones basadas en una sola condición, utilizando la estructura "Si-Entonces-Sino".
Crea ejercicios que requieran el uso de condiciones compuestas, utilizando los operadores lógicos.
Proporciona ejercicios que combinen la estructura "Si-Entonces-Sino" con bucles y otros conceptos aprendidos anteriormente.

#### Paso 4: Ejemplos de aplicaciones prácticas

Muestra ejemplos de aplicaciones prácticas de las estructuras de control condicionales en PSeint, como un programa que determine si un número es primo, o un programa que calcule el índice de masa corporal (IMC) basado en el peso y la altura de una persona.
Recuerda alentar a los participantes a comentar su código y explicar cada paso de sus algoritmos. Además, brinda retroalimentación y resuelve las dudas que puedan surgir durante los ejercicios prácticos. Al final del día, repasa los conceptos clave y asigna tareas de práctica adicional para reforzar el aprendizaje. ¡Buena suerte con el tercer día de curso!

## Contribución

¡Gracias por considerar contribuir a este proyecto! Si deseas agregar nuevos ejercicios que sean útiles para enseñar programación, sigue estos pasos:

1. Haz un fork de este repositorio y clónalo en tu máquina local.
2. Crea una nueva rama para tu contribución: `git checkout -b agregar-ejercicio`.
3. Agrega tus nuevos ejercicios en la carpeta correspondiente.
4. Realiza un commit de tus cambios: `git commit -m "Agregar nuevos ejercicios: nombre del ejercicio"`.
5. Envía tus cambios al repositorio remoto: `git push origin agregar-ejercicio`.
6. Abre una solicitud de extracción (pull request) para revisar tus cambios.
